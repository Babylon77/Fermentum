# Fermentum Security and Privacy

## Security principles

- least privilege
- secret isolation
- encrypted transport
- exportable user data
- no hidden cloud dependence for core workflows

## Client rules

- no embedded production secrets
- sensitive tokens stored with platform-secure storage where available
- local database encryption evaluated before synced release

## Server rules

- keep AI provider keys server-side
- audit privileged operations
- log security events without leaking sensitive payloads

## Privacy rules

- local-only mode is first-class
- synced features are opt-in
- AI use must be transparent
- user data retention needs explicit policy before public launch
